## Support Scope

ROBOTIS provides technical support for AntBot's **hardware and officially provided software**.

| Supported | Not Supported |
| :--- | :--- |
| Hardware defects and malfunctions | User-written custom software |
| Official firmware and updates | Third-party sensor/device integration |
| Drive system (Dynamixel motors) | General ROS 2 usage questions |
| Sensor hardware (LiDAR, cameras) | SLAM/Navigation parameter tuning |
| Battery and charging system | Issues caused by unofficial OS/kernel changes |
| RCU board and HMI panel | Network infrastructure configuration |

:::caution
Technical support is **not available for modified or customized hardware**. This includes frame modifications, unofficial parts replacement, and electrical circuit alterations.
:::

## Paid Technical Support

If you require professional engineering services beyond the standard support scope, paid technical support is available.

| Service | Description |
| :--- | :--- |

| On-site Visit | On-site engineer dispatch support |

For details on paid support, please contact us using the information below.

## Issue Handling Procedure

1. Check the [Known Issues](/antbot/en/maintenance/known-issues/) page to see if the same issue has already been reported.
2. Review the [Error Codes](/antbot/en/maintenance/error-codes/) page for the cause and recommended actions for your error code.
3. If the issue remains unresolved, please submit it through the methods below.

### GitHub Issues

Software bug reports and feature suggestions can be submitted through GitHub Issues.

- [Submit an Issue](https://github.com/ROBOTIS-move/antbot/issues/new)

### Community Resources

For software-related questions outside the support scope, such as general ROS 2 usage or SLAM/Navigation tuning, please refer to the following communities.

- [ROS Discourse](https://discourse.ros.org/)
- [ROBOTIS Community](https://community.robotis.com/)
